# 4. 브랜치 정책

| 배포환경 |    브런치명    |
| -------- | :------------: |
| main     |      런칭      |
| stage    | 주단위 작업물  |
| dev      |  실시간 개발   |
| PID-n    | Jira 티켓 번호 |

> 예시) PID/{작업명}

---

# 5. 커밋 정책

| 앱       | 종류                              |
| -------- | --------------------------------- |
| feat     | 새기능 추가                       |
| fix      | 버그 수정                         |
| format   | 앱에 영향을 미치지 않는 단순 수정 |
| refactor | 리팩토링                          |

> 예시) feat: 로그인 추가

# 6. PR 정책

1. 동일 스택 개발자 모두 요청
2. 모두의 approve 필요
3. 파트장이 merge 실행
